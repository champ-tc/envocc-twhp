Locking rule: คำตอบที่เป็น finished ล็อกถาวร — แม้แต่ ODPC แก้ไม่ได้ Sticky approvals carry over ทุกรอบ
Re-submit gate: โรงงานส่ง Cover ได้ก็ต่อเมื่อไม่มีคำตอบใดเป็น rejected — ต้อง accept หรือ redo ทุกรายการก่อน ถ้า Cover เป็น finished ส่งใหม่ไม่ได้
Race-free design: Tier-1 ไม่เขียน coverLogs เลย มีเพียง ODPC เท่านั้นที่เปลี่ยนสถานะ Cover — จึงไม่มี race ระหว่างผู้ประเมินหลายคนหรือระหว่างโรงงานกับผู้ประเมิน
Admin = ODPC ระดับประเทศ (ADR-3): ผู้ดูแลระบบ (role DOED) เข้าร่วมขั้น ODPC ได้ผ่าน GET/POST /admin/covers/{coverId}/{answers,verdict} (adminGuard) — ระบบสร้าง context { level: "ODPC", region: null } ให้อัตโนมัติ จึงมีอำนาจเท่า ODPC ทุกประการแต่ไม่จำกัด region (เห็น/ปิดงาน Cover ได้ทุกเขต) · audit ด้วย accountId ของแอดมินในคอลัมน์เดิม (ไม่เปลี่ยน schema)
